  rationale:

    HTML::Tabulate takes a table definition and a dataset (e.g. an
    arrayref of arrayrefs/hashrefs, DBIx::Recordset object, etc.) and
    combines them to display the dataset in an HTML table. The table
    definition allows precise control over all HTML table tags and
    attributes, and supports features like automatic striping, arbitrary
    cell formatting, automatic link creation, etc.

    HTML::Table is similar in spirit but has a more verbose, general,
    and interactive interface, where HTML::Tabulate is terser, more
    'batch-oriented', and better at working with complex datasets.
