my plans:

Within my thesis I've got to solve the problem of data presentation from
information system. The developed module should be used by information
system end-users (i.e. secretairies, teachers, etc.) and so no direct SQL
inputs are allowed and security mechanisms should be insured. The solutions
I've tested like AxKit or Cocoon taglibs and XSQL have usually used SQL in
their XML templates and therefore I haven't chosen them.

The module should process XML templates and it will act as XML::SAX filter
processing certain namespaces (XSLT is no option for end-users).

Planned namespace of the module is "Presentator".
