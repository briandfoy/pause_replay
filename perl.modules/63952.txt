I wrote a perl wrapper around the Java HtmlUnit library, which I
currently have up on CPAN as just "HtmlUnit". I was thinking that a
slightly better name might be WWW::HtmlUnit. What do y'all think?

In case you unfamiliar with HtmlUnit, http://htmlunit.sourceforge.net/
says 'HtmlUnit is a "browser for Java programs". It models HTML
documents and provides an API that allows you to invoke pages, fill out
forms, click links, etc... just like you do in your "normal" browser.'

In other words, it is similar to WWW::Mechanize (but adds javascript
support and some other things).
