I saw, recently, that I somehow had permissions on Config::Ini.  I don't know
when that happened.  That's not my point, though.

I uploaded a new Config-INI, with Config::INI.  Then I saw that I had
permissions to both ::Ini and ::INI.  So, I thought, "huh... how many other
modules have this potential problem?"

It turns out that 368 do.  I've attached my list, generated by the attached
script and 06perms.txt.

It seems like this is a problem that could bite someone using a
case-insensitive filesystem.  I know we can't get the genie back in the bottle,
but we could prevent case-varied versions of new module permissions from being
granted, and we could do something to clean up as many existing cases as
possible.  Steffen's purge of registered-and-not-used namespaces may help.

while (<>) { last if $_ eq "\n" }

my %first;
my %saw;
while (<>) {
  chomp;
  my ($module) = split /,/;

  next if $saw{$module}++;

  $first{lc $module} ||= $module;

  print "$module vs $first{lc $module}\n" if $module ne $first{lc $module};
}
