So I wrote an email-sending script which, from my limited testing, seems
to work fine. It sends a mail to $USERID@cpan.org asking about a
specific namespace registration and proposing a deletion three months in
the future if the author doesn't object.

The most difficult part would now be to actually remove any reasonably
recent registrations from the list of modules-without-upload. Given a
reduced list of that sort, I would suggest putting that list into a
place which only the PAUSE admins can edit. Then, we can remove
namespaces from it as authors complain about the scheduled deletion.

I'm inlining the script below. It also includes a rough cut at the email
which would be sent. And I'll include a full list as written by the
preparation script below the mailing script, too. Comments welcome! (Or
rather: required)

Best regards,
Steffen

P.S.: The program takes a file name as argument with "Name::Space
AUTHORID" lines.

#!/usr/bin/perl
use strict;
use warnings;

use Email::Send;
use DateTime;
use DateTime::Duration;

my $message = <<'HERE';
To: foo@foo.foo
From: modules@perl.org
Subject: THIS WILL BE REPLACED

Dear CPAN contributor,

we, the PAUSE administrators, are trying to tidy up the CPAN Modules
List and are looking at namespaces that have been registered without
ever being uploaded to PAUSE.

You have registered the namespace

NAMESPACE

in the CPAN Modules List, but according to our databases, you have not
uploaded any module with that name. This may be an error on our part,
a problem with the PAUSE indexer, or you actually haven't uploaded an
implementation.

In any case, we kindly ask you to contact us via the modules@perl.org
mailing list so we can sort out the issue. This is particularly
important if you think you are receiving this mail in error!

A simple message telling us briefly why you want to keep the
registration even without an upload so far should suffice to get the
namespace off our to-do list.

We plan to purge the registration for the aforementioned module
from our database somewhen after DATE if we do not receive an
answer until then.

Thank you for contributing to CPAN and your help in advance.

Best regards,
Steffen Mueller
for the PAUSE administrators
HERE

my $sender = Email::Send->new({
  mailer => 'SMTP',
  mailer_args => [Host => 'smtp.example.com'],
});

#for ( qw[SMTP Sendmail Qmail] ) {
#  $sender->mailer($_) and last if $sender->mailer_available($_);
#}

$sender->message_modifier(
  sub {
    my ($sender, $message, $author, $namespace) = @_;
    $message->header_set(To => "$author\@cpan.org");
    $message->header_set(Subject => "Old CPAN namespace registration for
$namespace");

    # insert namespace placeholder
    my $body = $message->body();
    $body =~ s/NAMESPACE/$namespace/g;

    # insert deletion-date placeholder
    my $date = DateTime->now() + DateTime::Duration->new(months => 3);
    my $datestr = $date->month_name . " " . $date->day_of_month . ", " .
$date->year;
    $body =~ s/DATE/$datestr/g;

    $message->body_set($body);
  },
);

while (<ARGV>) {
  chomp;
  my ($namespace, $author) = split /\s+/, $_, 3;
  my $result = $sender->send($message, $author, $namespace);
  if ($result =~ /Message sent/) {
    print "Sent mail to $author about $namespace.\n";
  }
  else {
    print "WARNING: Sending mail to $author about $namespace failed.
Reason: $result\n";
    print "Hit <Enter> to continue\n";
    my $tmp = <STDIN>;
  }
}



__END__
