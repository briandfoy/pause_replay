Seconded, thirded, whatever.

The US_DOD::* submissions must not be accepted.

The only use for documents and standards I can see when the document
contains some sort of structured data that can be programmatically
used, like for example the ISO 3166 country codes by Locale::Country.
There is a mapping from country code to the country name, and vice
versa, and programs can use that.

Wholesale dumping of standards and other documents (freeform text) is
not what CPAN is for, podified or not.  What next?  The RFCs?  The W3C
specs?  The ECMA standards?  The Bible in 791 languages?
