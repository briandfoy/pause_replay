I would like to have a module of mine, IniFile, currently in
    $CPAN/modules/by-authors/id/A/AV/AVATAR/
to be added to the module list and hence available from the
by-modules/by-category tree.  Here is a module description for your
consideration please.

Name              DSLI  Description                             Info
----------------  ----  --------------------------------------- -----
IniFile           RdpO  Accesses Windows .ini and .reg files    AVATAR

I think it should go to the "12_Opt_Arg_Param_Proc" category.

This modules provides easy access to MS-Windows style .ini files, Unreal
style extended .ini files (where multiple values can be associated with
a single key), as well as registry files with automatic conversion from
native Perl to Windows registry data encoding and vice versa.

IniFile was first released half a year ago and is used by UmodPack, an
Unreal mod installer:
    http://umodpack.sourceforge.net/

This module differs from another module, IniConf, in that this one
supports single-key-multiple-value, Windows registry file format and has
a different parser that may be a little more tolerant about wierd data.
