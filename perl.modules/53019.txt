  similar:
    DateTime::TimeZone::OlsonDB

  rationale:

    This module parses tzfile(5) files as generated by zic(8). These
    files encode arbitrary timezone rules, and are principally used with
    the Olson zoneinfo database. This module is in the form of a class
    implementing the DateTime::TimeZone interface, so it interoperates
    with DateTime.

    I published a development version of this module on the
    <datetime@perl.org> list. There were no objections to the concept,
    the code, or to it going on CPAN. There weren't any explicitly
    positive responses either; I'm taking silence to indicate assent.
    (The situation is identical to that with DT::TZ::SystemV.)

    DT::TZ::OlsonDB does a similar job, but it parses zoneinfo *source*
    files, skipping the tzfile(5) binary step. Of course, the sources
    are not commonly kept around, so parsing a binary found in the wild
    is a useful capability.
