<p><h3><a name="intro">Introduction</a></h3>
The Perl/Tk module list is here for the same reason as the main <a
href="http://www.perl.com/CPAN/modules/00modlist.long.html"> Perl module
list</a>, to change duplication of effort into cooperation and help to save
the world! Do your best to design such that your <b>Tk::*</b> module will make
good use of the building blocks found on <a
href="http://www.perl.com/CPAN">CPAN</a>, putting together new building blocks
where appropriate and simply glue them together with the Tk API. Hopefully
these modules will also serve as good examples. <a name="part"></a></p>

<h3>Playing Your Part</h3>
Tk and Perl are world-wide collaborative efforts, naturally, the Perl/Tk
integration project is the same.<br>
You've started on the right foot simply by reading this document, thanks!<br>
Please be sure you've read the main <a
href="http://www.perl.com/CPAN/modules/00modlist.long.html"> Perl module
list</a>, all that is said there is relevant here!

<p>If you have any additions or changes for the list, please send them to the
Perl/Tk <a href="mailto:ptk@lists.Stanford.EDU"> mailing list </a> or to <a
href="mailto:ach@mpe.mpg.de">me</a>.</p>

<p>If you wish to share your module(s) the Perl way, by putting them on <a
href="http://www.perl.com/CPAN">CPAN</a>, send a request to <a
href="mailto:modules@perl.com">modules@perl.com</a> to get set up with a PAUSE
user id and directory.</p>

<p><h3><a name="modules">The Perl/Tk modules</a></h3>
All the information corresponds to the latest updates I have received. We
don't record the version number or release dates of the listed Modules. Nor do
we record the locations of these Modules. Consult the contact, try the usual
perl CPAN sites or ask in <a
href="news:comp.lang.perl.tk">comp.lang.perl.tk</a> or on the Perl/Tk mailing
list <a href="mailto:ptk@lists.Stanford.EDU">&lt;ptk@lists.Stanford.EDU></a>
Please do *not* ask me directly, I simply don't have the time. Sorry.</p>

<p>Each Module listing is very short. The main goal is to simply publish the
existence of the modules, or ideas for modules, and enough contact information
for you to find out more. Each listing includes some characters which convey
(approximate) basic status information.</p>

<p></p>

<h4>For example:</h4>
<pre>Name           DSLI  Description                                   Info
-------------  ----  --------------------------------------------  -----
Cloth          Rdpo  Object interface to Tk::Canvas and items      GBARR
</pre>

<p></p>

<h4>Where the 'DSLI' characters have the following meanings:</h4>
<ul>
  <li>D - Development Stage  (Note: *NO IMPLIED TIMESCALES*):
    <ul>
      <li>i   - Idea, listed to gain consensus or as a placeholder</li>
      <li>c   - under construction but pre-alpha (not yet released)</li>
      <li>a/b - Alpha/Beta testing</li>
      <li>R   - Released</li>
      <li>M   - Mature (no rigorous definition)</li>
      <li>S   - Standard, supplied with Perl 5</li>
    </ul>
  </li>
</ul>
<ul>
  <li>S - Support Level:
    <ul>
      <li>m   - Mailing-list</li>
      <li>d   - Developer</li>
      <li>u   - Usenet newsgroup comp.lang.perl.tk</li>
      <li>n   - None known, try comp.lang.perl.tk or comp.lang.perl.misc</li>
    </ul>
  </li>
</ul>
<ul>
  <li>L - Language Used:
    <ul>
      <li>p   - Perl-only, no compiler needed, should be platform
      independent</li>
      <li>c   - C and perl, a C compiler will be needed</li>
      <li>+   - C++ and perl, a C++ compiler will be needed</li>
      <li>o   - perl and another language other than C or C++</li>
    </ul>
  </li>
</ul>
<ul>
  <li>I - Interface Style
    <ul>
      <li>f   - plain Functions, no references used</li>
      <li>r   - some use of unblessed References or ties</li>
      <li>O   - Object oriented using blessed references and/or
      inheritance</li>
    </ul>
  </li>
</ul>
Where letters are missing they can usually be inferred from the others.  For
example 'i' implies 'id', 'S' implies 'Su'.

<p>The Info column gives a contact reference 'tag'. Lookup this tag in the <a
href="#contacts">Contacts</a><br>
section of this document.  If no contact is given always try asking in <a
href="news:comp.lang.perl.tk">comp.lang.perl.tk</a>. or on the Perl/Tk mailing
list <a
href="mailto:ptk@lists.Stanford.EDU">&lt;ptk@lists.Stanford.EDU></a></p>

<p></p>

<h4>Ideas For Adoption</h4>
Modules listed as in the 'i' Development Stage with no contact reference are
ideas without an owner. Feel free to 'adopt' these but please let me know so
that we can update the list and thus inform anyone else who might be
interested. Adoption simply means that you either hope to implement the module
one day or would like to cooperate with anyone else who might be interested in
implementing it.

<p></p>

<h4>Cooperation</h4>
Similarly, if an idea that interests you has been adopted by someone please
contact them so you can share ideas.  Just because an idea has been adopted
does NOT imply that it's going to be implemented. Just because a module is
listed and being implemented does NOT mean it'll get finished. Waiting
silently in the hope that the Module will appear one day is unlikely to be
fruitful! Offer to help. Cooperate. Pool your efforts. Go on, try it!

<p>The same applies to modules in all states. Most modules are developed in
limited spare time. If you're interested in a module don't just wait for it to
happen, offer to help.</p>

<p>Module developers should feel free to announce incomplete work early. If
you're not going to be able to spend much time on something then say so. If
you invite cooperation maybe someone will implement it for you!</p>
