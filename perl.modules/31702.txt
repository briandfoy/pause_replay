Proposed module:

Font::Hershey    cdphp

   (Contains Font::Hershey::Occidental.pm
             Font::Hershey::Oriental.pm    (maybe ...)
             Font::Hershey::Tk.pm
   )

Basically yet another re-working of the venerable Hershey vector fonts.

One question I have for modules list is should this make vector-ness
more obvious? Font::Vector::Hershey ??

Bulk of module is the sub-package Font::Hershey::Occidental
which populates %glyph and %font from the "original"
 comp.sources.unix volume 4=20
version of the fonts.

The font hashes are indexed by (Unicode) character. I am woking on assignin=
g=20
Unicode code points to (most of) Hershey glyphs.

Hershey.pm provides an API :

sub render
{
 my ($face,$x,$y,$string,$callback,$xform) =3D @_;
 ...
}

$xform is optional and can rotate/scale glyphs to suid caller.

Callback is passed ref to array of vectors to render the string.
Hershey/Tk.pm provides interface to draw on Tk Canvas.

This provides the often requested ability to draw vertical/angled text
(and text also scales with rest of canvas items).
(Screen capture of a quick demo attached.)

Data and API are also useful in OpenGL=C2=AE applications - which is why it=
 seems=20
to make sense to put it in Font:: namespace rather than Tk::

Plan is to have C<render> call Unicode::Normalize::NFD=20
and "kern" accent glyphs onto base glyphs. Needs some new accent=20
glyphs for even iso-8859-1 - may be able to extract=20
those from GhostScript versions of Hershey fonts.
