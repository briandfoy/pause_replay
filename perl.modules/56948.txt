	I do agree that publishing multi-file scripts is not self-evident. But, if it 
can give you back some courage, I think you were real close to get it 
right. :-)

	I think that changing the NAME argument in Makefile.PL from 'tidyview.pl' 
to 'Perl::Tidyview' might be all what the indexer needs to know that the 
tarball is for this module. If it doesn't work, a foolproof way to get it 
working is to create a Perl::Tidyview module as part of the distribution. 

	As an example is worth a thousand words, I've attached to this email a 
modified version of the latest tidyview distro where I've both corrected the 
Makefile.PL, and created the eponymous module (in which I've moved the guts 
of the tidyview script).  I can't promise I didn't introduce bugs doing so, 
but the script comes up and seems to work without any visible hitch. 

	Btw, if I may suggest, the modules in the distro are a little spreaded out in 
term of namespace. To tighten things a little bit, I would rename them such 
that they all fit under the Perl::Tidyview umbrella:

	PerlTidy::Options	=>	Perl::Tidyview::Perltidy::Options
	PerlTidy::Run		=>	Perl::Tidyview::Perltidy::Run
	TidyView::Display	=>	Perl::Tidyview::Display
	TidyView::Frame		=>	Perl::Tidyview::Frame
	TidyView::Options	=>	Perl::Tidyview::Options
	TidyView::VERSION	=>	Perl::Tidyview::VERSION


	I hope this helps a little bit. If you have any further questions, please 
don't hesitate to poke be back.

	(oh, and serendipity had it that I discovered tidyview two weeks ago. This is 
an excellent application. I love it!)
